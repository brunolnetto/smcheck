## 2. State Variants

### 2.1 `State` (atomic or compound/parallel container)

**Constructor:**
```python
State(
    name: str = "",           # human-readable label
    value: Any = None,        # stored value (defaults to id string)
    initial: bool = False,    # Is this the initial state?
    final: bool = False,      # Is this a final state?
    parallel: bool = False,   # parallel region?
    states: List[State] = [], # child states (makes it compound/parallel)
    history: List[HistoryState] = [],  # history pseudo-states
    enter: callback_spec,     # inline enter callbacks
    exit: callback_spec,      # inline exit callbacks
    invoke: callback_spec,    # inline invoke callbacks
    donedata: callback_spec,  # only valid on final states; provides done.state data
)
```

**Derived properties:**
| Property | Meaning |
|---|---|
| `state.id` | Python attribute name (set by metaclass) |
| `state.name` | Human-readable name |
| `state.value` | Stored value in model |
| `state.initial` | Is initial? |
| `state.final` | Is final? |
| `state.parallel` | Is parallel region? |
| `state.is_atomic` | `bool(not state.states)` |
| `state.is_compound` | `bool(state.states) and not state.parallel` |
| `state.is_history` | `isinstance(state, HistoryState)` |
| `state.parent` | Parent `State` or `None` |
| `state.states` | Child `State` list |
| `state.history` | `HistoryState` pseudo-states |
| `state.transitions` | `TransitionList` of outgoing transitions |
| `state.document_order` | Integer set by metaclass for SCXML ordering |

**Instance proxy (`InstanceState`):**
Available at runtime via `sm.<state_id>`. Adds:
- `is_active` ÔÇö `True` if `state.value in sm.configuration_values`

### 2.2 `State.Compound` (class-body syntax)

Metaclass `NestedStateFactory` converts a class body into a compound `State`:

```python
class MyState(State.Compound):
    child_a = State(initial=True)
    child_b = State(final=True)
    go = child_a.to(child_b)  # TransitionList assigned to event
    
    def on_enter_child_a(self):  # method callbacks inline
        ...
```

All `State` instances in the class body become children. `TransitionList` values become events. Callable values become callbacks on the compound state.

### 2.3 `State.Parallel` (class-body syntax)

Same as `State.Compound` but with `parallel=True`. All children are entered simultaneously:

```python
class Region(State.Parallel):
    track_a = State.Compound(...)
    track_b = State.Compound(...)
```

### 2.4 `HistoryState`

```python
HistoryState(
    name: str = "",
    value: Any = None,
    type: HistoryType | str = HistoryType.SHALLOW,
)
```

**`HistoryType`** (`str` Enum):
- `HistoryType.SHALLOW` (`"shallow"`) ÔÇö remembers only direct children
- `HistoryType.DEEP` (`"deep"`) ÔÇö remembers full leaf path

History states are added to a compound state's `history` list. On entry, the engine resolves through `get_effective_target_states()`:
1. If `state.id in sm.history_values` Ôćĺ restore saved states
2. Otherwise Ôćĺ follow default transition on the history pseudo-state

**History is saved in `_prepare_exit_states()`:**
- **Shallow**: `[s for s in sm.configuration if s.parent == compound_state]`
- **Deep**: `[s for s in sm.configuration if s.is_descendant(compound_state)]`

### 2.5 `AnyState`

Internal pseudo-state used with `from_.any()`:

```python
go = closed.from_.any()  # one transition from EVERY non-final state
```

The `_on_event_defined` hook clones the transition for each non-final state in the machine.

## 3. Transition Variants and Attributes

### 3.1 `Transition` constructor

```python
Transition(
    source: State,
    target: State | List[State] | None,  # None = targetless
    event: str | List[str] | None,
    internal: bool = False,
    initial: bool = False,              # marks synthetic "enter initial" transitions
    validators: callback_spec = None,
    cond: callback_spec = None,         # must evaluate to True
    unless: callback_spec = None,       # must evaluate to False
    on: callback_spec = None,
    before: callback_spec = None,
    after: callback_spec = None,
)
```

**Key attributes:**
| Attribute | Type | Meaning |
|---|---|---|
| `transition.source` | `State` | Origin state |
| `transition.target` | `State \| None` | First target (for multi-target) |
| `transition.targets` | `List[State]` | ALL targets (SCXML parallel entry) |
| `transition.internal` | `bool` | Internal = no exit/enter actions on common ancestor |
| `transition.initial` | `bool` | Synthetic initial transition |
| `transition.is_self` | `bool` | `target is source` |
| `transition.is_eventless` | `bool` | No events attached |
| `transition.cond` | `SpecListGrouper` | Guards (must be `True`) |
| `transition.validators` | `SpecListGrouper` | Validators (raise to block) |
| `transition.before` | `SpecListGrouper` | Before callbacks |
| `transition.on` | `SpecListGrouper` | On-transition callbacks |
| `transition.after` | `SpecListGrouper` | After callbacks |
| `transition._events` | `Events` | Bound event names |
| `transition.event` | `str` | String representation of events |

### 3.2 Transition types by target

| Type | How declared | `target` | `internal` | Description |
|---|---|---|---|---|
| Regular | `a.to(b)` | `b` | `False` | Full exit/enter cycle |
| Self-transition | `a.to(a)` or `a.to.itself()` | `a` | `False` | Exit + re-enter `a` |
| Internal | `a.to(a, internal=True)` | `a` | `True` | No exit/enter (SCXML internal) |
| Targetless | `Transition(a, None)` | `None` | any | No state change |
| Multi-target | `source.to([b, c])` | `b` (first) | `False` | Enters multiple states (parallel) |
| Eventless | No event assigned | any | any | Fires on null event (immediate) |
| Wildcard | event `"*"` | any | any | Matches any event |

### 3.3 `TransitionList`

Container of `Transition` objects. Supports:
- `list1 | list2` ÔÇö combine into new `TransitionList`
- `@event.before`, `@event.after`, `@event.on`, `@event.cond`, `@event.unless` ÔÇö decorator syntax to add callbacks to all contained transitions
- `.unique_events` ÔÇö list of unique `Event`s across all transitions
- `.has_eventless_transition` ÔÇö bool property

## 4. Callback / Hook Naming Convention System

### 4.1 `CallbackGroup` enum (execution order in a microstep)

```
PREPARE Ôćĺ ENTER Ôćĺ EXIT Ôćĺ INVOKE Ôćĺ VALIDATOR Ôćĺ BEFORE Ôćĺ ON Ôćĺ AFTER Ôćĺ COND
```

Actual execution order per microstep:
1. **`prepare`** (`prepare_event`) ÔÇö runs before ANY transition, returns new `kwargs`
2. **`validators`** ÔÇö raise to block the transition (no guard logic)
3. **`cond/unless`** ÔÇö evaluated; must all pass for transition to be selected
4. **`before`** ÔÇö runs before exit/enter actions
5. **`exit`** ÔÇö state exit callbacks (reversed document order)
6. **`on`** ÔÇö transition content callbacks
7. **`enter`** ÔÇö state entry callbacks (forward document order)
8. **`after`** ÔÇö runs after enter actions and state config update
9. **`invoke`** ÔÇö spawned after macrostep completes

### 4.2 Convention-based hooks (auto-registered by `_setup()`)

**State hooks** (on State object `_setup()`):
```
on_enter_state          # generic enter hook (ALL states)
on_enter_{state_id}     # state-specific enter hook
on_exit_state           # generic exit hook (ALL states)
on_exit_{state_id}      # state-specific exit hook
on_invoke_state         # generic invoke hook (ALL states)
on_invoke_{state_id}    # state-specific invoke hook
```

**Transition hooks** (on Transition object `_setup()`):
```
before_transition           # generic before hook (ALL transitions)
before_{event_id}           # event-specific before hook
on_transition               # generic on hook (ALL transitions)
on_{event_id}               # event-specific on hook
after_{event_id}            # event-specific after hook
after_transition            # generic after hook (ALL transitions) ÔÇö runs LAST
```

**Machine-level hooks:**
```
prepare_event               # runs before EVERY transition (can inject kwargs)
```

### 4.3 Priority order (lower = earlier in `insort`)

```python
class CallbackPriority(IntEnum):
    GENERIC   = 0   # on_enter_state, before_transition, etc.
    INLINE    = 10  # declared in State/Transition constructor kwargs
    DECORATOR = 20  # @state.enter, @event.before, etc.
    NAMING    = 30  # on_enter_{id}, before_{event}, on_{event}, after_{event}
    AFTER     = 40  # after_transition (always last)
```

### 4.4 `callback_spec` accepted types

All callback positions accept:
- `str` ÔÇö resolved by name from the SM instance, model, or listeners
- `callable` ÔÇö unbounded method or lambda
- `property` ÔÇö resolved on the SM/model instance
- `List[str | callable | property]` ÔÇö multiple callbacks applied in order

### 4.5 Decorator syntax

```python
# On events (TransitionList):
@sm_class.event_name.before
def my_before(self): ...

@sm_class.event_name.after
def my_after(self): ...

@sm_class.event_name.on
def my_on(self): ...

@sm_class.event_name.cond         # expected_value=True
def my_cond(self) -> bool: ...

@sm_class.event_name.unless       # expected_value=False
def my_unless(self) -> bool: ...

@sm_class.event_name.validators
def my_validator(self): ...       # raise to block

# On states:
@state.enter
def my_enter(self): ...

@state.exit
def my_exit(self): ...
```

## 5. Guard / Condition Mechanisms

### 5.1 `cond=` parameter

- Accepts: string name, callable, or list of either
- ALL conditions must be `True` for the transition to be selected
- Evaluated using `callbacks.all()` (short-circuit AND)

### 5.2 `unless=` parameter

- Creates a `cond` entry with `expected_value=False`
- "Not X" ÔÇö transition fires only when `unless` returns `False`

### 5.3 `validators=` parameter

- Run before conditions
- Can **raise** an exception to block the transition
- Does NOT return a boolean ÔÇö purely side-effect; if it raises, transition fails
- `on_error=None` (errors propagate, not caught as events)

### 5.4 Boolean expression strings (spec_parser)

When a `cond=` argument is a string, it is parsed as a boolean expression supporting:
```
!x          Ôćĺ not x
x ^ y       Ôćĺ x and y
x v y       Ôćĺ x or y
x and y     Ôćĺ python AND (native)
x or y      Ôćĺ python OR (native)
x == y      Ôćĺ equality comparison
x != y, x > y, x >= y, x < y, x <= y
in("state_id", ...)    Ôćĺ built-in: checks if all named states are in sm.configuration
```

Short-circuit evaluation is preserved (incl. async).

### 5.5 Filter-by-derivation conflict resolution

When multiple transitions from different states in the current configuration are enabled simultaneously, the engine applies SCXML conflict resolution:
- Prefer transitions from descendant states over ancestor states
- Among same-level states, prefer document order

## 6. Action / Callback Passing System

### 6.1 Keyword arguments injected into every callback

The `EventData.extended_kwargs` dict always contains:
```python
{
    "event_data": EventData,     # full event data object
    "machine": StateChart,       # the SM instance
    "event": str,                # event name
    "model": Any,                # the domain model
    "transition": Transition,    # current transition
    "state": State,              # current state during callback
    "source": State,             # origin state
    "target": State,             # destination state
    **kwargs,                    # user-supplied event kwargs
}
```

The `SignatureAdapter` automatically selects which parameters each callback needs ÔÇö no need to accept all kwargs.

### 6.2 `prepare_event` hook

- Runs before any transition content
- Can **return a dict** to inject additional kwargs for all subsequent callbacks

### 6.3 Positional argument threading

Event calls accept `*args`. These flow through as `EventData.args` and are passed positionally to callbacks.

## 7. Event Features

### 7.1 `Event` class

`Event` is a subclass of `str` (the event ID is the string value). Key attributes:
```python
Event(
    transitions: str | Transition | TransitionList | None,
    id: str | None,
    name: str | None,
    delay: float = 0,    # milliseconds before processing
    internal: bool = False,  # placed on internal queue
)
```

### 7.2 Calling events

```python
sm.my_event()                    # sync fire
sm.my_event(arg1, kwarg=val)     # with args
sm.send("my_event", **kwargs)    # string-based fire
sm.raise_("my_event", **kwargs)  # internal queue (processed first)
sm.my_event.put(...)             # nonblocking put (no wait for result)
```

### 7.3 Delayed events

```python
sm.send("event_name", delay=3000)        # 3-second delay in ms
sm.send("event_name", delay=500, send_id="timer1")
sm.cancel_event("timer1")               # cancel before firing
```

### 7.4 Wildcard matching

`Event.match(event_str)` rules:
- `"*"` matches ANY event
- Prefix match: `"error"` matches `"error"`, `"error.execution"`, `"error.foo.bar"`
- Trailing `.*` or `.` stripped before prefix check

### 7.5 Special event naming conventions (SCXML)

`_expand_event_id(key)` transforms attribute names:
```
done_invoke_<id>  Ôćĺ  "done_invoke_<id> done.invoke.<id>"
done_state_<id>   Ôćĺ  "done_state_<id> done.state.<id>"
error_<x>         Ôćĺ  "error_<x> error.<x>"    (underscore Ôćĺ dot)
```

So `done.invoke.<id>` and `done_invoke_<id>` are both registered.

### 7.6 `done.invoke.<state_id>` (automatic)

When an `invoke` handler's `run()` completes (or `InvokeGroup` finishes), the engine queues `done.invoke.<state_id>` on the internal queue.

### 7.7 `done.state.<state_id>` (automatic)

When all children of a compound state have reached their final state, the engine queues `done.state.<parent_id>` on the internal queue.

### 7.8 `error.execution` (automatic)

When `catch_errors_as_events=True` and a callback raises, `error.execution` is placed on the internal queue with `error=<exception>` in kwargs.

## 8. Internal vs External Transitions

| Dimension | External (default) | Internal |
|---|---|---|
| Declaration | `a.to(a)` | `a.to(a, internal=True)` |
| `transition.internal` | `False` | `True` |
| Exit actions | Fired | NOT fired |
| Enter actions | Fired | NOT fired |
| Self-restriction | Any target | Target must be `self` or a descendant |
| `enable_self_transition_entries` | N/A | N/A (controls only self-transitions in `StateMachine`) |

In `StateChart`: `enable_self_transition_entries=True` means self-transitions DO execute enter/exit.
In `StateMachine`: `enable_self_transition_entries=False` (legacy compatibility).

## 10. Multiple Conditions (AND / OR logic)

### AND (default list behavior):
```python
t = a.to(b, cond=["is_ready", "has_permission"])  # ALL must be True
```

### OR via boolean expression string:
```python
t = a.to(b, cond="is_ready v has_permission")  # at least one True
```

### Combined:
```python
t = a.to(b, cond=["is_valid ^ has_stock", "not_cancelled"])
```

### `unless=` negation:
```python
t = a.to(b, unless="is_locked")  # fires if is_locked returns False
```

Multiple `unless` entries are each individually negated and AND-ed together with any `cond` entries.

## 11. Exception / Error Handling in Transitions

### `catch_errors_as_events=True` (SCXML mode, default in `StateChart`)

Any runtime exception in callbacks:
1. Does NOT propagate to the caller
2. Queues `error.execution` internal event with `error=<exception>` in kwargs
3. If already processing `error.execution`, the new error is logged and dropped (infinite-loop guard)

### `catch_errors_as_events=False` (`StateMachine` default)

Exceptions propagate normally. External queue is cleared on uncaught exceptions.

### `validators=` flow:
- Validators raise ÔÇö transition is blocked, exception propagates (regardless of `catch_errors_as_events`)

## 12. History States (Shallow vs Deep)

```python
class Door(State.Compound):
    open = State(initial=True)
    closed = State()
    history = HistoryState()                    # shallow (default)
    deep_history = HistoryState(type="deep")    # deep
```

**Resolution in `get_effective_target_states()`:**
1. Check `sm.history_values[history.id]`
2. If found Ôćĺ restore those states
3. If NOT found Ôćĺ follow the default transitions declared on the `HistoryState` itself

**Saving (in `_prepare_exit_states()`):**
- Shallow: `[s for s in sm.configuration if s.parent == parent_compound]`
- Deep: `[s for s in sm.configuration if s.is_descendant(parent_compound)]`

## 13. Compound State Entry/Exit Semantics

### Entry:
1. Metaclass creates default `initial=True` transitions for the first child in document order
2. On entry, engine calls `compute_entry_set()` which:
   - Adds the target state and all its ancestors (up to LCCA) to the entry set
   - For compound states: also adds the initial child and its descendants
   - For parallel states: adds ALL children and their initial descendants
3. Callbacks fire in document order (ascending)

### Exit:
1. Exit set computed via `_compute_exit_set()` using Least Common Compound Ancestor (LCCA)
2. All active states in the exit set are exited in reversed document order (deepest first)
3. `_prepare_exit_states()` saves history BEFORE firing exit callbacks

### Invoke lifecycle:
- `on_invoke_{state_id}` is called AFTER entering states (post-macrostep)
- Invocations are cancelled (via `ctx.cancelled.set()`) BEFORE firing exit callbacks

## 14. Parallel State Completion Conditions

A compound parallel state `P` fires `done.state.P` when:
- ALL atomic descendants that are children of `P`'s parallel sub-regions have reached their `final` state

Engine check in `_handle_final_state()`:
```
if target.parent is not None:
    parent = target.parent
    grandparent = parent.parent
    if grandparent and grandparent.parallel:
        if all(child.final and child in sm.configuration for child in grandparent.states[...]):
            Ôćĺ queue done.state.<grandparent.id>
```

## 15. Engines

### 15.1 `SyncEngine` (default when no async callbacks)

- `processing_loop()` ÔÇö synchronous, uses a `threading.Lock` for re-entrancy
- Re-entrant calls: event put on queue, processed sequentially by current holder
- Invoke handlers run in daemon **threads** via `ThreadPoolExecutor`
- Delayed events: busy-loop polling with `time.sleep(sm._loop_sleep_in_ms)` (default 1ms)
- Returns first result of first external event processed

### 15.2 `AsyncEngine` (auto-selected when any async callback exists)

- `processing_loop()` ÔÇö async coroutine
- Re-entrancy detection via `contextvars.ContextVar(_in_processing_loop)`
- External events get an `asyncio.Future` for `await`-able result
- Invoke handlers run as `asyncio.Task`s
- Uses `asyncio.gather` for concurrent callback dispatch

### 15.3 Auto-selection

```python
def _get_engine(self):
    if self._callbacks.has_async_callbacks:
        return AsyncEngine(self)
    return SyncEngine(self)
```

### 15.4 Event queues (both engines)

- **External queue**: `PriorityQueue` sorted by `TriggerData.execution_time`
- **Internal queue**: `PriorityQueue` for `raise_()` / `error.execution` / `done.*`
- Internal events are processed BEFORE external events in each macrostep cycle

### 15.5 Processing algorithm (simplified)

```
loop:
  1. Drain internal queue & eventless transitions (microsteps)
  2. Spawn pending invoke handlers
  3. Check root final state Ôćĺ stop if reached
  4. Pop from external queue
     - If delayed, re-queue and sleep
     - Otherwise: select_transitions Ôćĺ microstep
  5. repeat while events remain
```

## 16. The `contrib/` Module

### 16.1 `contrib/diagram.py` ÔÇö `DotGraphMachine`

Generates a `pydot` graph from a `StateChart` class or instance.

```python
from statemachine.contrib.diagram import DotGraphMachine

graph = DotGraphMachine(MyMachine).get_graph()
graph.create_svg()      # SVG bytes
graph.create_png()      # PNG bytes (requires graphviz installed)
graph.create_dot()      # DOT source
```

Features:
- Active states highlighted (turquoise, penwidth=2)
- Compound states Ôćĺ `pydot.Subgraph` clusters
- Parallel states Ôćĺ dashed borders
- History pseudo-states shown as `H` or `H*` circles
- State labels include entry/exit/internal action annotations
- Transition labels: `event_name\n[cond1, cond2]`
- Respects `lhead`/`ltail` for cluster edges
- Class-level customizable: `graph_rankdir`, `font_name`, `state_font_size`, `transition_font_size`, `state_active_penwidth`, `state_active_fillcolor`

### 16.2 `contrib/timeout.py` ÔÇö `timeout()`

```python
from statemachine.contrib.timeout import timeout

waiting = State(initial=True, invoke=timeout(5.0, on="expired"))
```

- Returns a `_Timeout` implementing `IInvoke.run(ctx)`
- Waits `duration` seconds; if state not exited Ôćĺ sends `on` event (or auto `done.invoke.*`)
- Uses `ctx.cancelled.wait(timeout=duration)` (cooperative, not busy-wait)
- Raises `ValueError` for non-positive duration

### 16.3 `contrib/weighted.py` ÔÇö `weighted_transitions()`

```python
from statemachine.contrib.weighted import weighted_transitions, to

move = weighted_transitions(
    standing,
    (walk, 70),                           # plain (state, weight) tuple
    to(run, 30, cond="has_energy"),       # with extra kwargs
    seed=42,                              # optional RNG seed
)
```

- Index 0 calls `random.choices()` with `weights` list; others check cached result
- `_WeightedGroup` holds `random.Random(seed)` for reproducibility
- `_make_weighted_cond(index, group, weight, total)` returns a closure with `__name__ = "weight=30 (30%)"`
- Conditions added at `CallbackPriority.GENERIC` with `expected_value=True`

## 20. Metaclass Validators (`StateMachineMetaclass`)

The metaclass runs these checks during **class construction**:

| Flag | Check | Error |
|---|---|---|
| `validate_disconnected_states=True` | BFS reachability from initial state | `InvalidDefinition: unreachable states` |
| `validate_trap_states=True` | Non-final states with no outgoing transitions | `InvalidDefinition: trap states` |
| `validate_final_reachability=True` | Non-final states with no path to any final | `InvalidDefinition: no path to final` |
| Always | Exactly one initial state at top level | `InvalidDefinition: initial state count` |
| Always | Final states have no outgoing transitions | `InvalidDefinition: transition from final` |
| Always | Initial transitions have no conditions or events | `InvalidDefinition: initial transition` |
| Always | Internal transitions: target must be self or descendant | `InvalidDefinition: invalid internal` |
| Always | `donedata` only on final states | `InvalidDefinition: donedata on non-final` |

All checks raise `InvalidDefinition` (subclass of `StateMachineError`).

## 21. Execution Model Summary

### Macrostep / Microstep model

- **Macrostep**: triggered by one external event
- **Microstep**: one atomic transition (or set of parallel transitions)
  - Calls: `before` Ôćĺ `exit` Ôćĺ `on` Ôćĺ `enter` Ôćĺ `after`
- **Null event cycle**: eventless transitions and internal events are processed between external events

### `StateChart` class-level configuration flags

| Attribute | Default | Effect |
|---|---|---|
| `allow_event_without_transition` | `True` | If `False`, unknown/unmatched events raise `TransitionNotAllowed` |
| `enable_self_transition_entries` | `True` | If `False`, self-transitions skip enter/exit |
| `atomic_configuration_update` | `False` | If `True`, configuration updated atomically after all callbacks |
| `catch_errors_as_events` | `True` | If `True`, callback errors Ôćĺ `error.execution` event |
| `_loop_sleep_in_ms` | `0.001` | Sleep duration between delayed event polls |
| `start_configuration_values` | `[]` | Override initial configuration |
| `start_value` | `None` | Constructor param ÔÇö start at specific state value |

### Serialization

`StateChart` supports pickle:
- `__getstate__` excludes `_callbacks`, `_states_for_instance`, `_engine`
- `__setstate__` rebuilds all three plus re-registers listeners

## 23. `invoke` Feature (State-level Invocations)

### Declare

```python
loading = State(initial=True, invoke=my_callable)
loading = State(initial=True, invoke=[callable1, callable2])        # sequential
loading = State(initial=True, invoke=invoke_group(fn1, fn2))        # concurrent
loading = State(initial=True, invoke=ChildMachine)                  # child SM
loading = State(initial=True, invoke=timeout(5.0, on="expired"))    # timeout
```

### Convention callbacks

```
on_invoke_state         # generic invoke hook
on_invoke_{state_id}    # state-specific invoke hook
```

### `IInvoke` protocol

```python
class MyInvoker:
    def run(self, ctx: InvokeContext) -> Any: ...
    def on_cancel(self): ...  # optional cleanup
```

### `InvokeContext` attributes

```python
ctx.invokeid   # str: unique invocation id
ctx.state_id   # str: the state that triggered this invocation
ctx.send       # callable: send(event_name, **kwargs) on parent machine
ctx.machine    # StateChart: the parent machine
ctx.cancelled  # threading.Event: set when state exits
ctx.kwargs     # dict: event kwargs at state entry time
```

### `invoke_group(*callables)` ÔÇö concurrent execution

- Wraps callables in `ThreadPoolExecutor`
- All must complete; any exception Ôćĺ `error.execution`
- Returns list of results; accessible in `done.invoke` callbacks via `data=`

### `StateChartInvoker` ÔÇö child state machine

- Instantiates child SM and runs it synchronously
- `on_cancel()` stops child engine and cancels its invocations
- `done.invoke.<state_id>` fired when child's `is_terminated` becomes `True`

## 27. What `smcheck` Must Handle That Simple FSMs Don't

1. **Hierarchical (compound) states** ÔÇö transitions from a child may be inherited from ancestors; graph traversal must walk state hierarchy
2. **Parallel regions** ÔÇö configuration is a SET, not a single state; all parallel leaves are simultaneously active
3. **History pseudo-states** ÔÇö restore previous configuration on re-entry
4. **Eventless transitions** ÔÇö fire automatically (no event trigger); create implicit arcs in the graph
5. **Internal transitions** ÔÇö don't cause enter/exit; may not change the "active state" visible externally
6. **Multi-target transitions** ÔÇö one transition enters multiple states (parallel)
7. **`done.state.*` / `done.invoke.*` events** ÔÇö auto-generated, not declared explicitly by user
8. **`error.execution` event** ÔÇö implicit; fires on callback errors in SCXML mode
9. **Guard boolean expressions** ÔÇö string cond can contain `and`, `or`, `not`, `in()`, comparisons
10. **Weighted transitions** ÔÇö probabilistic selection via `cond` closures; graph analysis should treat all targets as reachable
11. **Invoke** ÔÇö invocations spawn background work and may send events; creates temporal ordering constraints
12. **`AnyState` / wildcard** ÔÇö `from_.any()` creates N copies of a transition (one per non-final state)
13. **Wildcard events** ÔÇö `"*"` matches any event; prefix matching for `"error.*"`
14. **`prepare_event`** ÔÇö can inject kwargs that affect all callbacks in a microstep
15. **Serialization** ÔÇö SM can be pickled/unpickled; validators and graph structure don't change
